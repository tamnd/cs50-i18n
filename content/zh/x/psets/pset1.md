## 做什么

1. 使用你的 GitHub 账户登录 [submit.cs50.io](https://submit.cs50.io)，点击 **Authorize cs50**，然后关闭该标签页。
2. 使用你的 GitHub 账户登录 [cs50.dev](https://cs50.dev/) 以访问你专属的 “codespace”。
3. codespace 加载完成后，关闭默认打开的所有 **Welcome** 标签页。
4. 在 codespace 的终端窗口运行 `update50` 以确保你的 codespace 是最新的，如果出现提示，点击 **Rebuild now**。
5. 完成 [Hello, World](world/)。
6. 提交 [Hello, It’s Me](me/)。
7. 提交以下之一：
   
   - 如果觉得不太自信，提交 [这个版本的 Mario](mario/less/)
   - 如果觉得比较自信，提交 [这个版本的 Mario](mario/more/)
8. 提交以下之一：
   
   - 如果觉得不太自信，提交 [Cash](cash/)
   - 如果觉得比较自信，提交 [Credit](credit/)

如果你提交了两个版本的 Mario，我们将记录两个分数中的较高者。如果你同时提交了 Cash 和 Credit，我们将记录两个分数中的较高者。

## 建议

- 通过 [第 1 周](../../weeks/1/) 的源代码尝试 David 在课堂上的任何程序。
- 查看 CS50 的 [C 语言代码风格指南](https://cs50.readthedocs.io/style/c/)，获取有关如何改进代码风格的建议。
- 如果在使用 `make` 编译代码时看到任何错误，请首先专注于修复看到的第一个错误，并根据需要向上滚动。如果不确定错误的含义，请尝试向 `help50` 寻求帮助。例如，如果尝试编译 `hello` 时，
  
  ```
  make hello
  ```
  
  产生错误，请尝试运行
  
  ```
  help50 make hello
  ```
  
  代替！
