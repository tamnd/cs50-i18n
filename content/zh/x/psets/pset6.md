关于习题集的合作是不允许的，除非你可以向同学和其他人寻求帮助，但只要这种帮助不会沦为他人替你完成工作，根据课程的 [学术诚实](../../syllabus/#academic-honesty) 政策。

工作人员会对提交给 CS50x 的作品进行随机审计。被发现违反此政策的学生将被移出课程。已经完成 CS50x 的学生，如果被发现违规，其 CS50 证书将被永久撤销。

## 该做什么

1. 使用你的 GitHub 账号登录 [cs50.dev](https://cs50.dev)
2. 运行你的 codespace 终端窗口中的 `update50` 以确保你的 codespace 是最新的。
3. 提交 Python 版的 [Hello](hello/)
4. 提交以下之一：
   
   - 如果你觉得基础不够扎实，提交 Python 版的 [此版本 Mario](mario/less/)
   - 如果你觉得基础比较扎实，提交 Python 版的 [此版本 Mario](mario/more/)
5. 提交以下之一：
   
   - 如果你觉得基础不够扎实，提交 Python 版的 [Cash](cash/)
   - 如果你觉得基础比较扎实，提交 Python 版的 [Credit](credit/)
6. 提交 Python 版的 [Readability](readability/)
7. 提交 Python 版的 [DNA](dna/)

如果你提交了两个版本的 Mario，我们将记录两个分数中的最高分。如果你同时提交了 Cash 和 Credit，我们将记录两个分数中的最高分。

## 学术诚实

- 对于 Hello、Mario、Cash、Credit 和 Readability，查看你自己以及他人的 *C 语言* 实现是 **合理的**。
- 查看他人对应的 *Python* 实现是 **不合理的**。
- 既然这些题目的目标之一是教会你如何自学一门新语言，请记住根据教学大纲，以下行为不仅是 **合理的**，而且是为了实现该目标而鼓励的：
  
  - 将你在网上或其他地方找到的几行代码合并到你自己的代码中，前提是这些代码行本身不是指定问题的解决方案，并且你注明了代码行的来源。
  - 向网络或其他地方寻求课程之外的指导、参考资料以及技术难题的解决方案，但不能寻找习题集问题的直接解决方案或你自己的最终项目。
