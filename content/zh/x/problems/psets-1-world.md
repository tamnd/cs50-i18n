## 待解决的问题

感谢 Brian Kernighan 教授（David 上学时教他 CS50 的老师！），“hello, world” 已经用数百种语言实现了。让我们把你的实现也加入到这个列表中吧！

在一个名为 `world` 的文件夹中创建一个名为 `hello.c` 的文件，用 C 语言实现一个程序，只打印 `hello, world\n` 即可！

提示

这就是你应该编写的实际代码！（挺像样的一个提示，对吧？）不过，最好还是亲手输入，而不是复制/粘贴，这样你就能开始培养编写代码的“肌肉记忆”。

```c
#include <stdio.h>

int main(void)
{
    printf("hello, world\n");
}
```

## 如何开始

在终端窗口中单独执行 `cd`。你会发现终端窗口的提示符类似于：

```
$
```

接下来执行

```bash
mkdir world
```

在你的 codespace 中创建一个名为 `world` 的文件夹。

然后执行

```bash
cd world
```

进入该文件夹。你现在应该看到终端提示符为 `world/ $`。现在你可以执行

```
code hello.c
```

创建一个名为 `hello.c` 的文件，并在其中编写代码。

## 如何测试

请记住，你可以使用以下命令编译 `hello.c`：

```
make hello
```

如果你没有看到错误消息，说明编译成功了！你可以通过以下命令进行确认：

```bash
ls
```

它不仅应该列出 `hello.c`（源代码），还应该列出 `hello`（机器码）。

如果你确实看到了错误消息，请尝试修复代码并再次尝试编译。如果你不理解错误消息，请尝试执行：

```
help50 make hello
```

以获取建议。

一旦你的代码编译成功，你就可以通过以下命令执行程序：

```
./hello
```

### 正确性

执行以下命令，使用 `check50` 评估代码的正确性。这是一个命令行程序，每当你的代码通过 CS50 的自动测试时，它就会输出笑脸；如果没有通过，它就会输出哭脸！

```
check50 cs50/problems/2026/x/world
```

### 风格

执行以下命令，使用 `style50` 评估代码风格。这是一个命令行程序，它会输出你应该对程序进行的修改，包括添加（绿色）和删除（红色），以改进其风格。如果你看这些颜色有困难，`style50` 也支持其他[模式](https://cs50.readthedocs.io/style50/)！

```
style50 hello.c
```
