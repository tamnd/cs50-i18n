問題セットでの協力は禁止されています。ただし、クラスメートや他の人に助けを求めることは、その助けが他人に自分の課題を代わりにやってもらうことにならない限り、コースの[学術的誠実さ](../../syllabus/#academic-honesty)に関するポリシーに従って許可されます。

スタッフはCS50xの提出物に対してランダムな監査を行います。このポリシーに違反したことが判明した学生は、コースから除名されます。すでにCS50xを修了している学生が違反していることが判明した場合、その学生のCS50修了証は永久に剥奪されます。

## やること

1. **Chrome、Edge、またはSafariを使用し**（Firefoxは不可）、GitHubアカウントを使用して[cs50.dev](https://cs50.dev)にログインします。
2. codespaceのターミナルウィンドウで `update50` を実行して、codespaceが最新であることを確認します。プロンプトが表示されたら、**Rebuild now**をクリックします。
3. [Scrabble](scrabble/) を提出します。
4. [Readability](readability/) を提出します。
5. 次のいずれか1つを提出します。
   
   - 自信がない場合は、[Caesar](caesar/)
   - 自信がある場合は、[Substitution](substitution/)

CaesarとSubstitutionの両方を提出した場合、2つのスコアのうち高い方が記録されます。

## 提出期限

[2026-12-31T23:59:00+00:00](https://time.cs50.io/20261231T235900Z) まで。

## アドバイス

- [Week 2](../../weeks/2/)の例を通して、授業で紹介されたDavidのプログラムを試してみてください。
- C 関数のマニュアルページを見るには、[manual.cs50.io](https://manual.cs50.io/) にアクセスしてください。
- `make` でコードをコンパイルしたときにエラーが表示された場合は、必要に応じて上にスクロールして、最初に表示されたエラーの修正に集中してください。意味がわからない場合は、`help50` に助けを求めてみてください。たとえば、`readability` をコンパイルしようとして、
  
  ```
  make readability
  ```
  
  でエラーが発生している場合は、代わりに
  
  ```
  help50 make readability
  ```
  
  を実行してみてください！

## 助けを求める方法

- コースの[コミュニティ](../../communities/)で助けを求めてください！
